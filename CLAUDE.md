## Project Overview

**2026-05-GameJam** is a Unity 2D game project built with Unity 6000.4.8f1 (Unity 6). It uses the Universal Render Pipeline (URP) with the 2D Renderer.

## Game Design

### 장르 및 조작
- **장르**: 횡스크롤 액션 게임 (Side-scrolling action)
- **조작**: 키보드 기반 액션. 기술적 문제가 생기면 점프 액션만으로 축소 가능
- **아트 스타일**: 픽셀 아트 권장 (아트 담당자 재량)

### 스토리 개요
마피아 조직 **해달 패밀리**의 일원인 수달 **칼립소(Calypso)**가 마지막 임무를 마치고 보스에게 은퇴를 선언하며 시작.

- **튜토리얼**: 은퇴를 축하하러 온 조직원들을 제거 대상으로 오해하는 상황. 아무도 해치지 않으면 추가 컷씬 분기.
- **본편**: 신흥 조직 **비버번(Beaverbun)**이 칼립소 영입 실패 후 위험 요소로 판단해 습격. 살아남은 칼립소가 복수를 다짐하고, 해달 패밀리의 지원을 받아 비버번의 계획을 저지.

### 등장인물

## Input System

The input asset is `Assets/InputSystem_Actions.inputactions` with two action maps:

**Player map** — control scheme bindings:
| Action | Keyboard&Mouse | Gamepad |
|--------|---------------|---------|
| Move | WASD / Arrow keys | Left Stick |
| Look | Mouse delta | Right Stick |
| Attack | Left Mouse / Enter | Button West |
| Jump | Space | Button South |
| Sprint | Left Shift | Left Stick Press |
| Interact | E (Hold) | Button North (Hold) |
| Crouch | C | Button East |
| Previous | 1 | DPad Left |
| Next | 2 | DPad Right |

**UI map** — standard Unity UI navigation (Navigate, Submit, Cancel, Point, Click, ScrollWheel, etc.)

Control schemes: `Keyboard&Mouse`, `Gamepad`, `Touch`, `Joystick`, `XR`

## Rendering Setup

- Pipeline: URP 2D — configured in `Assets/Settings/UniversalRP.asset` and `Assets/Settings/Renderer2D.asset`
- Scene template: `Assets/Settings/Lit2DSceneTemplate.scenetemplate`
- Default resolution: 1920×1080
- Color space: Linear

## Asset Folder Conventions

```
Assets/
  Animations/   # Animator controllers and animation clips
  Images/       # Sprites, textures, tilesets
  Prefabs/      # Reusable prefabs
  Scenes/       # Unity scenes (SampleScene.unity is the default)
  Scripts/      # C# MonoBehaviour and other scripts
  Settings/     # URP pipeline and renderer assets
  Sounds/       # Audio clips
```

## Working with Scripts

After creating or modifying any `.cs` file via MCP tools, compilation is triggered automatically — wait for `is_compiling == false` in `editor_state` before attaching components or running play mode.

Scripts go in `Assets/Scripts/`. When referencing the Input System in code, the generated C# class from `InputSystem_Actions.inputactions` is `InputSystem_Actions` (auto-generated by Unity).

## Running Tests

Use Unity Test Framework via MCP:
```python
run_tests(mode="EditMode")   # or "PlayMode"
get_test_job(job_id=..., wait_timeout=60, include_failed_tests=True)
```
